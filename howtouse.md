# คู่มือหลัง Clone โปรเจกต์ (Camera Rental System)

เอกสารนี้สำหรับเพื่อนที่ **clone รีโป้จาก GitHub** แล้วต้องการ **รันโปรเจกต์ต่อ** หรือ **พัฒนาต่อ** ให้ทำตามลำดับด้านล่าง

**สำคัญ:** รีโป้ถูกตั้งค่าให้ **ไม่ commit ไฟล์ลับ / ของที่ generate บนเครื่อง** (เช่น `node_modules`, `.env`, โฟลเดอร์อัปโหลดรูป) — พอ clone มาเลยจะ **ไม่มีไฟล์พวกนี้** นั่นเป็นพฤติกรรมปกติ ต้อง **สร้างใหม่บนเครื่องตัวเอง** ตามคู่มือด้านล่าง

---

## 1) สิ่งที่ต้องติดตั้งบนเครื่องก่อน

- **Node.js** (แนะนำ LTS)
- **PostgreSQL** (เช่น 15+) และถ้าต้องการดูข้อมูลในฐานข้อมูลแบบ GUI ใช้ **pgAdmin4**
- **Git**

---

## 2) Clone และติดตั้ง dependency (`node_modules`)

ที่โฟลเดอร์ที่ต้องการเก็บโปรเจกต์:

```bash
git clone <URL-รีโป้ของคุณ>
cd cp252project
npm install
```

### ทำไมไม่มีโฟลเดอร์ `node_modules` หลัง clone?

- โฟลเดอร์ **`node_modules/`** ถูกใส่ใน **`.gitignore`** เพราะใหญ่มาก และแต่ละ OS อาจ build ต่างกัน — **จะไม่ถูก push ขึ้น GitHub**
- หลัง clone **ต้องรัน `npm install` ทุกครั้ง** (หรือเมื่อ `package.json` เปลี่ยน) เพื่อดึงแพ็กเกจมาไว้ในเครื่องตัวเอง
- ถ้ามีโฟลเดอร์ย่อย `CameraRentalSystem/node_modules` เก่าๆ ในเครื่อง — โปรเจกต์นี้ใช้ **`package.json` ที่รากรีโป้** เป็นหลัก; การติดตั้งที่รากจะพอสำหรับรัน `npm start`

> แพ็กเกจติดตั้งที่ **รากโปรเจกต์** (`package.json` อยู่ที่ราก) ไม่ใช่แค่ใน `CameraRentalSystem/`

## 3) ไฟล์ที่ “เพิ่งลบออกจาก Git / ไม่เคยอยู่ใน Git” — อธิบายและวิธีสร้างใหม่

ทีมมักใช้ commit แบบ **chore: remove sensitive files and node_modules from git track** เพื่อให้รีโป้สะอาดและปลอดภัย ผลคือเพื่อนที่ clone จะ **ไม่ได้** ไฟล์กลุ่มนี้จาก Git — ต้องสร้างเองตามตาราง

| สิ่งที่ไม่อยู่ใน Git | เหตุผล | วิธีสร้าง / ทดแทนบนเครื่องเพื่อน |
|---------------------|---------|-----------------------------------|
| **`node_modules/`** | ขนาดใหญ่, generate จาก `package.json` | รัน **`npm install`** ที่รากรีโป้ (ดูข้อ 2) |
| **`.env`** | มีรหัส DB, session secret — **ห้าม commit** | คัดลอกจาก **`.env.example`** แล้วแก้ค่า (ดูข้อ 4) |
| **ไฟล์ `.env` อื่นๆ** | `.gitignore` กันไว้ทั้งหมด ยกเว้น `.env.example` | ใช้แค่ `.env` หนึ่งไฟล์ที่รากตามคู่มือ |
| **`CameraRentalSystem/public/uploads/`** (รูปสินค้า, สลิป, avatar ฯลฯ) | เป็นข้อมูลรันไทม์ของแต่ละเครื่อง / อาจมีข้อมูลส่วนบุคคล | **ไม่ต้อง copy มือ** — เมื่อรันแอป ฟังก์ชัน `ensureUploadDirectories()` ในโค้ดจะสร้างโฟลเดอร์ `public/uploads` และหมวด `products`, `website`, `slips`, `avatars`, `others` ให้อัตโนมัติ (ถ้ายังไม่มี) |
| **`database.sqlite` / `*.sqlite`** | ถ้าเคยมีในรีโป้เก่า มักถูกเอาออกหรือ ignore — แอปหลักใช้ **PostgreSQL** ไม่ใช่ SQLite | **ไม่ต้องสร้างไฟล์ sqlite** สำหรับรันแอปปัจจุบัน; ให้สร้างฐานข้อมูลใน PostgreSQL แล้วตั้งค่าใน `.env` (ดูข้อ 5) |
| **`coverage/`** (รายงาน test coverage) | generate จากคำสั่งทดสอบ | ถ้าต้องการ รัน **`npm run test:cov`** จะสร้างใหม่ได้ |

### สรุปสั้นๆ สำหรับเพื่อน

1. **`npm install`** → ได้ `node_modules`  
2. **สร้าง `.env` จาก `.env.example`** → ได้ค่าเชื่อม DB และ session  
3. **รัน `npm start` ครั้งแรก** → โฟลเดอร์อัปโหลดถูกสร้างให้ (ถ้ายังไม่มี)  
4. **อย่า commit `.env` หรือรูปสลิปจริง** ขึ้น Git — ใช้ `.gitignore` ตามที่ทีมตั้งไว้

## 4) ตั้งค่า environment (`.env`) — สร้างยังไงทีละขั้น

ไฟล์ **`.env` ไม่อยู่ใน Git** เพื่อนต้องสร้างเอง:

1. ที่ **รากโปรเจกต์** สร้างไฟล์ `.env` โดยคัดลอกจาก template:

**Windows (PowerShell / CMD):**

```bash
copy .env.example .env
```

**macOS / Linux:**

```bash
cp .env.example .env
```

2. เปิด `.env` ด้วย editor แล้วแก้ค่าอย่างน้อย:

- `DB_HOST`, `DB_PORT`, `DB_NAME`, `DB_USER`, `DB_PASSWORD` — ให้ตรงกับ PostgreSQL ที่สร้างไว้
- `SESSION_SECRET` — สตริงยาวสุ่ม (อย่าใช้ค่า placeholder ตอนขึ้น production จริง)
- `ADMIN_USERNAME`, `ADMIN_EMAIL`, `ADMIN_PASSWORD` — ใช้ตอน **รันครั้งแรก** เมื่อยัง **ไม่มี admin ในฐานข้อมูล** ระบบจะสร้าง admin คนแรกจาก env (รหัส admin ต้องยาวตามที่โค้ดกำหนดใน `adminSeed.js`)

3. (กรณีปรับ schema ครั้งใหญ่) อาจตั้ง `DB_SYNC_ALTER=true` ชั่วคราวตาม `CameraRentalSystem/README.md` แล้วปิดเป็น `false` หลัง schema นิ่ง

---

## 5) สร้างฐานข้อมูล PostgreSQL

1. เปิด PostgreSQL (หรือ pgAdmin) แล้ว **สร้าง database เปล่า** เช่น `camera_rental`
2. ให้ชื่อ database ใน `.env` (`DB_NAME`) **ตรงกับที่สร้าง**
3. จดค่า host, port, user, password ใส่ใน `.env`

แอปจะ sync ตารางตอน startup ผ่าน `schemaSync` — ไม่ต้อง import ไฟล์ `.sqlite` จากที่อื่น

## 6) รันแอปพลิเคชัน

จาก **รากโปรเจกต์**:

```bash
npm start
```

เปิดเบราว์เซอร์ที่ `http://localhost:3000` (หรือพอร์ตจาก `PORT` ใน `.env`)

ครั้งแรกที่รันสำเร็จ ระบบจะ:

- เตรียมโฟลเดอร์อัปโหลดใต้ `CameraRentalSystem/public/uploads/` (ถ้ายังไม่มี)
- sync schema / seed กล้องเริ่มต้น / seed admin (ตามเงื่อนไขในโค้ด)

## 8) ถ้าจะให้ AI (เช่น Cursor / ChatGPT) ช่วยอธิบายหรือทำต่อโปรเจกต์

**ให้ทำตามขั้นตอนนี้ทุกครั้งที่เริ่มคุยกับ AI:**

1. **บอก AI ให้อ่านไฟล์ `progress_note.md` ในรากโปรเจกต์ก่อน** — สรุปสถาปัตยกรรม, โฟลเดอร์จริง, ฟีเจอร์ที่มีแล้ว, อัปเดตล่าสุด (รวมบล็อก **11/04/2026** ว่าโค้ดถึงไหนและยังขาดอะไร)
2. ถ้าต้องการให้ช่วยเรื่อง “หลัง clone ไม่มีไฟล์ X” ให้บอกว่า **อ่าน `howtouse.md` ด้วย** (ไฟล์นี้)
3. หลัง AI อ่านแล้ว ถามเช่น “ตอนนี้ระบบมีฟีเจอร์อะไรบ้าง” หรือ “อยากต่อฟีเจอร์ X ต้องแตะไฟล์ไหน”
4. รันเทสหรือ lint: `npm test`, `npm run lint` (ดูใน `package.json`)

**ตัวอย่างข้อความที่ copy ไปให้ AI:**

```text
โปรเจกต์อยู่ที่รากรีโป้ cp252project กรุณาอ่านไฟล์ progress_note.md และ howtouse.md ก่อน แล้วช่วยอธิบายว่าตอนนี้ระบบมีฟีเจอร์อะไรบ้าง หลัง clone ต้องสร้างไฟล์อะไรบ้าง และถ้าฉันจะทำต่อต้องทำอะไร
```

### ถ้าเพื่อน “ไม่รู้อะไรเลย” — ใช้ prompt นี้ (copy วางเดียว)

ข้อความด้านล่างออกแบบให้ AI **พาเริ่มจากศูนย์**: อ่านเอกสารในรีโป้ก่อน แล้วบอกทีละขั้นว่าต้องติดตั้งอะไร / รันคำสั่งอะไร / สร้างไฟล์อะไร — เพื่อนแค่ทำตามและแปะ error ถ้าติด

```text
ฉันเป็นมือใหม่เกี่ยวกับโปรเจกต์นี้มาก ยังไม่รู้ว่าต้องติดตั้งอะไรบ้าง
กรุณาอ่านไฟล์ howtouse.md และ progress_note.md ในรากโปรเจกต์ก่อนทุกครั้ง

จากนั้นช่วยพาฉันทีละขั้นแบบละเอียด (อย่าข้ามขั้นตอน) เพื่อให้รันโปรเจกต์ Camera Rental System บนเครื่องฉันได้จริง:
1) ต้องมีโปรแกรมอะไรบ้าง (Node, PostgreSQL ฯลฯ)
2) หลัง clone แล้วรันคำสั่งอะไร
3) สร้าง database ใน PostgreSQL ยังไง (บอกแบบมือใหม่)
4) สร้างไฟล์ .env จาก .env.example และต้องแก้ค่าอะไรบ้าง
5) รัน npm start แล้วเช็กว่าสำเร็จยังไง

ฉันใช้ระบบปฏิบัติการ: [ใส่ Windows / macOS / Linux]
ถ้าฉันติด error ฉันจะ copy ข้อความ error มาให้ — ช่วยอธิบายภาษาไทยทีละอย่าง
```

**หมายเหตุสำคัญ (อ่านก่อนโยนให้ AI):**

- AI **ช่วยติดตั้งโปรแกรมบนเครื่องแทนไม่ได้** — เพื่อนต้องดาวน์โหลด Node / PostgreSQL และรันคำสั่งในเทอร์มินัลเอง
- ถ้า AI ไม่ได้อยู่ใน Cursor/โปรเจกต์เปิดอยู่ อาจอ่านไฟล์ในรีโป้ไม่ได้ — ให้เพื่อน **แนบเนื้อหา `howtouse.md` / `progress_note.md` หรือเปิดโปรเจกต์ใน Cursor** แล้วสั่งให้อ่านไฟล์
- เมื่อติดปัญหา **copy ข้อความ error ทั้งก้อน** มาให้ AI จะแก้ได้ตรงจุดกว่าอธิบายกว้างๆ

## 10) ปัญหาที่พบบ่อย

- **ไม่มี `node_modules` / import ไม่เจอ** — รัน `npm install` ที่รากรีโป้  
- **ไม่มี `.env`** — สร้างจาก `copy` / `cp` `.env.example` `.env` แล้วแก้ค่า  
- **แอปไม่ขึ้น / error `SESSION_SECRET`** — ต้องตั้งใน `.env`  
- **เชื่อม DB ไม่ได้** — ตรวจ PostgreSQL เปิดอยู่, ค่า `DB_*` ตรงกับ database ที่สร้าง  
- **อัปโหลดรูปแล้วไม่เห็น** — ตรวจว่ารันแอปแล้วโฟลเดอร์ `CameraRentalSystem/public/uploads/` ถูกสร้าง; อย่าลืมว่าไฟล์อัปโหลดเก่าของเครื่องคนอื่น **ไม่ได้มากับ Git**  
- **สมัครแล้วไม่เห็นใน pgAdmin** — ตรวจว่าแอปชี้ DB เดียวกับที่เปิดใน pgAdmin และกด Refresh ตาราง
