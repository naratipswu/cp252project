# คู่มือหลัง Clone โปรเจกต์ (Camera Rental System)

เอกสารนี้สำหรับเพื่อนที่ **clone รีโป้จาก GitHub** แล้วต้องการ **รันโปรเจกต์ต่อ** หรือ **พัฒนาต่อ** ให้ทำตามลำดับด้านล่าง

---

## 1) สิ่งที่ต้องติดตั้งบนเครื่องก่อน

- **Node.js** (แนะนำ LTS)
- **PostgreSQL** (เช่น 15+) และเครื่องมืออย่าง **pgAdmin4** (ถ้าต้องการดูข้อมูลในฐานข้อมูล)
- **Git**

---

## 2) Clone และติดตั้ง dependency

ที่โฟลเดอร์ที่ต้องการเก็บโปรเจกต์:

```bash
git clone <URL-รีโป้ของคุณ>
cd cp252project
npm install
```

> แพ็กเกจติดตั้งที่ **รากโปรเจกต์** (`package.json` อยู่ที่ราก) ไม่ใช่แค่ใน `CameraRentalSystem/`

---

## 3) สร้างฐานข้อมูล PostgreSQL

1. สร้าง database เปล่า เช่น `camera_rental` (ชื่ออะไรก็ได้ แต่ต้องตรงกับ `.env`)
2. จดไว้: host, port, user, password, ชื่อ database

---

## 4) ตั้งค่า environment (`.env`)

1. ที่ **รากโปรเจกต์** ให้มีไฟล์ `.env` (สามารถคัดลอกจาก `.env.example` แล้วแก้ค่า)

```bash
copy .env.example .env
```

(บน macOS/Linux ใช้ `cp .env.example .env`)

2. แก้ค่าอย่างน้อยดังนี้ให้ตรงกับ PostgreSQL ของคุณ:

- `DB_HOST`, `DB_PORT`, `DB_NAME`, `DB_USER`, `DB_PASSWORD`
- `SESSION_SECRET` — สตริงยาวสุ่ม **อย่าใช้ค่า default ตอนขึ้น production**
- `ADMIN_USERNAME`, `ADMIN_EMAIL`, `ADMIN_PASSWORD` — ใช้ตอน **รันครั้งแรก** ถ้ายัง **ไม่มี admin ในฐานข้อมูล** ระบบจะ seed admin คนแรกจากค่าเหล่านี้ (รหัส admin ต้องยาวอย่างน้อยตามที่โค้ดกำหนด)

3. (ถ้าเพิ่งปรับ schema ครั้งใหญ่) อาจตั้ง `DB_SYNC_ALTER=true` ชั่วคราวตามที่อธิบายใน `CameraRentalSystem/README.md` แล้วปิดกลับเป็น `false` หลัง schema นิ่ง

---

## 5) รันแอปพลิเคชัน

จาก **รากโปรเจกต์**:

```bash
npm start
```

จากนั้นเปิดเบราว์เซอร์ที่ `http://localhost:3000` (หรือพอร์ตที่ตั้งใน `PORT`)

---

## 6) เช็กว่าใช้งานได้

- เปิดหน้าแรก → สมัครสมาชิก / เข้าสู่ระบบ
- เข้า `/browse` ดูรายการกล้อง
- (ถ้าเป็น admin) เข้า `/admin` — ใช้บัญชีที่ seed จาก `ADMIN_*` หรือบัญชีที่ถูก promote เป็น admin

ถ้าต้องการรายละเอียดการเชื่อม pgAdmin หรือ query ตรวจข้อมูล ให้ดูใน **`progress_note.md`** ส่วนคู่มือ pgAdmin และ checklist SQL

---

## 7) ถ้าจะให้ AI (เช่น Cursor / ChatGPT) ช่วยอธิบายหรือทำต่อโปรเจกต์

**ให้ทำตามขั้นตอนนี้ทุกครั้งที่เริ่มคุยกับ AI เกี่ยวกับโปรเจกต์นี้:**

1. **บอก AI ให้อ่านไฟล์ `progress_note.md` ในรากโปรเจกต์ก่อน** — ไฟล์นี้สรุปสถาปัตยกรรม, โฟลเดอร์ที่ใช้จริง, ฟีเจอร์ที่มีแล้ว, อัปเดตล่าสุด (รวมถึงบล็อกวันที่ **11/04/2026** ที่สรุปว่าโค้ดมีอะไรถึงไหนแล้ว และยังขาดอะไร)
2. หลัง AI อ่านแล้ว ให้ถามเช่น “ตอนนี้ระบบมีฟีเจอร์อะไรบ้าง” หรือ “อยากต่อฟีเจอร์ X ต้องแตะไฟล์ไหน”
3. ถ้าจะรันเทสหรือ lint ให้ดูคำสั่งใน `package.json` (`npm test`, `npm run lint`)

**ตัวอย่างข้อความที่ copy ไปให้ AI:**

```text
โปรเจกต์อยู่ที่รากรีโป้ cp252project กรุณาอ่านไฟล์ progress_note.md ก่อน แล้วช่วยอธิบายว่าตอนนี้ระบบมีฟีเจอร์อะไรบ้าง และถ้าฉันจะทำต่อต้องทำอะไร
```

---

## 8) ไฟล์อ้างอิงเพิ่มเติม

| ไฟล์ | ใช้ทำอะไร |
|------|-----------|
| `progress_note.md` | เอกสารลึก + ประวัติอัปเดต + ช่องว่างที่ยังไม่ทำ — **ให้ AI อ่านอันนี้ก่อน** |
| `CameraRentalSystem/README.md` | วิธีรัน แบบย่อ เน้น Postgres |
| `finalrequirement.md` | requirement เดิมของบางงาน (บางข้ออาจไม่ตรงพฤติกรรมโค้ดปัจจุบัน — ดูคำอธิบายใน `progress_note.md` บล็อก 11/04/2026) |
| `.env.example` | template ตัวแปรสภาพแวดล้อม |

---

## 9) ปัญหาที่พบบ่อย

- **แอปไม่ขึ้น / error เรื่อง `SESSION_SECRET`** — ต้องตั้งใน `.env` ที่รากโปรเจกต์
- **เชื่อม DB ไม่ได้** — ตรวจว่า PostgreSQL เปิดอยู่, `DB_*` ตรงกับที่สร้างจริง, ชื่อ database ที่สร้างใน pgAdmin ตรงกับ `DB_NAME`
- **สมัครแล้วไม่เห็นใน pgAdmin** — ตรวจว่าแอปรัน process เดียวกับที่ตั้งค่า `.env` และ refresh ตารางใน pgAdmin

---

## 10) สรุปสั้นๆ

1. Clone → `npm install` ที่รากรีโป้  
2. สร้าง DB PostgreSQL → สร้าง `.env` จาก `.env.example`  
3. `npm start` → เปิด `http://localhost:3000`  
4. **ให้ AI อ่าน `progress_note.md` ก่อน** เพื่อให้อธิบายสถานะโปรเจกต์และจุดงานต่อได้ตรงกับโค้ดจริง
